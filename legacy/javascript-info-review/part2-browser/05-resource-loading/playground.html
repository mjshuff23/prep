3. async vs defer (concept demo)

— <script> blocks parsing right there; DOM below is invisible to it — <script defer> downloads in parallel, runs AFTER parsing, IN ORDER, before DOMContentLoaded — <script async> downloads in parallel, runs WHENEVER READY, order = luck, DCL doesn't wait — type="module" defer semantics by default (async attr opts into async) Rule: defer for app scripts that need each other; async for independents (analytics/ads).
